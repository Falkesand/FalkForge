# FalkForge v0.5.0-beta.4

This is the fourth public beta. Where beta.3 turned several accepted-but-not-yet-real inputs into
working features, beta.4 is a security-and-honesty beta: it closes the one residual gap flagged in
the beta.3 notes (the download URL of an external container wasn't covered by the bundle's
signature envelope), adds a simpler and stronger way to sign a bundle, and fixes a real localization
bug where dialog titles and static UI text shipped an unresolved token into compiled MSIs instead of
the translated string. A handful of honesty and maintainability fixes round out the release, in the
same spirit as beta.2 and beta.3.

## Shipped this beta

- **External-container download URL is now bound into the ECDSA signature — closes the SSRF
  residual noted in beta.3.** Previously only per-package payload hashes were signed; a tampered
  bundle could repoint `Container.DownloadUrl` to an internal host without invalidating the
  signature. The download URL, its declared hash, and its container membership are now part of the
  signed envelope, so a tampered URL is rejected the same way a tampered payload hash always was.
- **Whole-bundle signing: the fully-assembled bundle `.exe` can now be signed directly**, and it
  still self-extracts correctly afterward. This is the recommended signing path going forward — one
  `signtool` call against the finished file, no intermediate `.stub.exe`/`.data` files, no TOC
  offset patching — and unlike the existing detach/sign/reattach fallback it does not rely on the
  CVE-2013-3900 certificate-padding leniency, so it also passes on machines with the opt-in strict
  certificate-padding hardening enabled. `BundleReader`'s footer lookup now falls back to the PE
  Security data directory when the FALKBUNDLE footer isn't at the physical end of the file (which it
  no longer is once a certificate table is appended after it). Detach/reattach is documented as the
  remaining choice only when the stub must be signed by a process that never sees the payloads (e.g.
  a remote signing service, or a pipeline where signing happens before payloads are known).
- **Fixed a real bug: dialog titles and static UI text (`UIText` rows) shipped the raw `!(loc.*)`
  token unresolved into compiled MSIs.** `!(loc.*)` resolution previously only covered
  `Control.Text`; it now also resolves dialog titles and `UIText`, so installers with per-culture
  string tables actually show translated titles and text instead of the literal placeholder. This
  also enables generating a proper per-culture MST for these strings.
- **Delta bundles fail loud on a corrupt delta blob instead of crashing.** An unattended updater
  applying a delta patch now gets a clear failure it can react to (fall back to a full download,
  alert, retry) rather than an unhandled crash — closes part of the delta test-debt (C15) noted
  after the delta-runtime-gap fix. New tests cover reconstruction plus the integrity-failure modes.
- **The decompiler now round-trips external downloadable containers correctly** when reconstructing
  a bundle from a built `.exe`.
- **`FalkOutputType=Msix` (the MSBuild SDK property) now fails loud instead of silently no-op-ing**,
  matching the `forge build --format msix` behavior already fixed in beta.3. The CLI's own MSIX
  message is now accurate: MSIX authoring is script-only today via `InstallerMsix.BuildMsix()` /
  `InstallerMsix.BuildMsixBundle()`, not through `forge build` or the SDK.
- **Honesty fix:** the `AcmeSuite` "real install" test no longer silently passes without exercising
  anything — it now either runs for real under a proper gate or reports an accurate skip.
- **`forge init` documentation gap fixed** — points newcomers at the getting-started tutorial
  accurately.
- **`*.mst` files are now gitignored** (generated per-culture language transforms, not source).
- **Housekeeping:** decomposed several oversized files for maintainability —
  `MsiRecipeBuilder`, `MsiPackageReconstructor`, `CabinetBuilder`, `DialogSetProducer`,
  `IisCommandFactory`, and `MsiAuthoring` are now split into focused partial-class files with the
  same public surface.
- **Added a real-machine verification runbook and script** (`docs/testing/real-machine-verification.md`)
  covering the beta.3/beta.4 live paths that only run under `FALKFORGE_REAL_SYSTEM_E2E=1` on an
  elevated machine.

## Known limitations / in progress

- **The live paths (real bundle install/rollback, IIS apply, feature-picker `ADDLOCAL`, external-
  container network download, turnkey UI, per-culture install UI) are built and unit-proven but
  only exercised end-to-end under `FALKFORGE_REAL_SYSTEM_E2E=1`** on a real, elevated machine — see
  `docs/testing/real-machine-verification.md` for the runbook. None of these run in CI.
- **MSIX build is not available via `forge build` or the MSBuild SDK** — it's script-only today,
  via `InstallerMsix.BuildMsix()` / `InstallerMsix.BuildMsixBundle()` called directly from a build
  script. Both entry points now fail loud (rather than silently falling back to an MSI) if you try
  to reach MSIX through them.

## Requirements

- Windows, to *build* MSI/MSM/MSP/MST output (P/Invokes `msi.dll`). Bundle compilation and the
  decompiler's bundle path are cross-platform; MSI compilation, inspection, and the WiX-Burn
  decompiler are Windows-only.
- .NET 10 SDK.
