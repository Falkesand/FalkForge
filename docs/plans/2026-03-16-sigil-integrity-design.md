# Sigil Integrity & SBOM Integration Design

## Goal

Add build-time payload signing and SBOM generation to FalkForge using Sigil.Sign, with runtime integrity verification in the engine. Proves at install time that every file is bit-for-bit what the developer packaged.

## Key Design Decisions

1. **Opportunistic signing** — Only signs if `sigil` CLI is found on PATH at build time. No signature = no verification at runtime (backward compatible). No build error if Sigil is absent.

2. **Ephemeral keys by default** — Zero-config tamper detection using throwaway ECDSA-P256 keys. Developers can optionally configure a persistent key (PEM file, Windows cert store thumbprint, or vault reference) for authorship proof.

3. **Embedded storage** — Manifest signature and SBOM stored inside the compiled MSI (custom table `_FalkForgeIntegrity`) or bundle (new `IntegritySection` in footer). Also written to disk as sidecar files for CI/CD.

4. **No Sigil dependency at runtime** — The engine verifies using built-in .NET SHA-256 hashing and ECDSA signature verification. End users never need Sigil installed.

5. **Fatal on tamper** — If a signature is present and verification fails, the engine refuses to install. No bypass, no override.

6. **SBOM dual format** — SPDX (default) or CycloneDX, developer's choice. Wrapped in Sigil DSSE attestation.

7. **Three SBOM access paths** — Bundle: `MyInstaller.exe --sbom report.json`. MSI: `forge inspect --sbom output.msi`. CI/CD: sidecar build output file.

## Architecture

### Build-Time Flow

```
Compile → Hash all payload files → sigil sign-manifest (with ephemeral or configured key)
       → Generate SBOM JSON (SPDX or CycloneDX)
       → sigil attest (wrap SBOM in DSSE attestation)
       → Embed manifest signature + SBOM attestation in output
       → Write sidecar files to disk
```

### Runtime Flow

```
Extract payloads → Read embedded IntegrityManifest
                 → If absent: proceed normally (unsigned, backward compatible)
                 → If present: verify ECDSA signature on manifest using embedded public key
                 → Hash each extracted file, compare against manifest entries
                 → All match: log "Integrity verified: N files" → continue to Planning
                 → Any mismatch: Fatal error INT002 → refuse to install
```

### Sigil Detection

At the start of compilation, the compiler calls `sigil --version` and caches the result. If the tool isn't found, all signing/SBOM steps are silently skipped (warning SGN001 logged).

## Fluent API

### MSI — PackageBuilder

```csharp
Installer.Build(args, package =>
{
    package.Name = "MyApp";
    // ... normal setup ...

    // Optional: configure signing key (ephemeral if omitted)
    package.Integrity(integrity => integrity
        .SigningKey("path/to/key.pem")           // or .CertStore("thumbprint") or .Vault("azure", "key-ref")
        .Sbom(SbomFormat.Spdx)                   // default; or .Sbom(SbomFormat.CycloneDx)
    );
});
```

### Bundle — BundleBuilder

```csharp
Installer.BuildBundle(args, bundle =>
{
    bundle.Integrity(integrity => integrity
        .SigningKey("path/to/key.pem")
        .Sbom(SbomFormat.CycloneDx)
    );
    // ...
});
```

### Default behavior (no .Integrity() call)

If Sigil is on PATH: auto-sign with ephemeral key, generate SPDX SBOM.
If Sigil is not on PATH: skip silently. No error.

### CLI override

`forge build myapp.csx --no-sign` — explicitly skip signing even when Sigil is available.

## Storage Format

### MSI — Custom table `_FalkForgeIntegrity`

| Column | Type | Content |
|--------|------|---------|
| `Id` | String (PK) | `ManifestSignature` or `SbomAttestation` |
| `Format` | String | `sigil-manifest-v1` or `spdx` / `cyclonedx` |
| `Data` | String | The JSON content |

Two rows. Standard MSI table readable by any MSI inspection tool.

### Bundle — IntegritySection in footer

Current format: `[PE stub][Magic][Manifest][Compressed payloads][TOC][Footer]`

Extended: `[PE stub][Magic][Manifest][Compressed payloads][TOC][IntegritySection][Footer]`

IntegritySection layout:
- 4-byte magic: `FINT`
- 4-byte manifest signature length
- Manifest signature JSON bytes
- 4-byte SBOM attestation length
- SBOM attestation JSON bytes

Footer offset table updated to include integrity section offset. Zero offset = unsigned (backward compatible).

### MSIX

SBOM attestation embedded as package asset file (`sbom.json`). No separate integrity manifest — MSIX uses its own Authenticode signing.

## SBOM Generation

Generated by FalkForge, not Sigil. Sigil only signs the attestation envelope.

### Per-file metadata

- File name and relative path
- SHA-256 hash
- File size
- Version (from PE version info if available)

### Package-level metadata

- Package name, version, manufacturer
- Build timestamp
- FalkForge version
- Target architecture

## SBOM Access

| Method | Audience | How |
|--------|----------|-----|
| `MyInstaller.exe --sbom report.json` | IT departments, end users | Bundle extracts embedded SBOM to file, exits without installing |
| `forge inspect --sbom output.msi` | Developers with CLI | Reads `_FalkForgeIntegrity` table, writes SBOM to stdout |
| `output.msi.sbom.json` (sidecar) | CI/CD pipelines | Build output file alongside the installer |

For the bundle `--sbom` flag: this is an early-exit path in `EngineHost`. Parse args, find `--sbom`, extract SBOM from IntegritySection, write to file, exit code 0. No install occurs.

If no SBOM embedded: print "No SBOM available in this installer", exit code 1.

## Error Codes

### Build-time (compiler)

| Code | Severity | Condition |
|------|----------|-----------|
| SGN001 | Warning | Sigil not found on PATH — signing skipped |
| SGN002 | Error | Signing key file not found at configured path |
| SGN003 | Error | `sigil sign-manifest` exited with non-zero code |
| SGN004 | Error | `sigil attest` exited with non-zero code |
| SGN005 | Warning | SBOM generation failed — build continues without SBOM |

### Runtime (engine)

| Code | Severity | Condition |
|------|----------|-----------|
| INT001 | Fatal | Integrity manifest signature is invalid (tampered hash list) |
| INT002 | Fatal | File hash mismatch — `{filename}` expected `{expected}` got `{actual}` |
| INT003 | Warning | Integrity section present but public key missing — cannot verify |

Fatal errors halt the engine. No override. No bypass.

## Security Properties

- **Tamper detection**: Any modification to payload files after build is detected at install time
- **No runtime tool dependency**: Verification uses built-in .NET cryptography
- **Backward compatible**: Unsigned installers work normally
- **Forward compatible**: New footer section is ignored by old engines (offset = 0)
- **Key compromise scope**: Ephemeral keys mean each build has a unique key — compromising one doesn't affect others
- **SBOM provenance**: DSSE attestation cryptographically ties the SBOM to the build
